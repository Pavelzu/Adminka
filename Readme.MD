Проект призван упростить жизнь 1ой линии ТП и автоматизировать некоторые рабочие моменты. 
Глобально функционал состоит из трех частей: 
1. Работа с LM-manager
Актуальность этого пункта будет сохраняться до тех пор, пока LightManager, установленный на компы пользаков, будет глючить и не позволять проходить удаленным подключениям. Функционал достаточно прост: указав доменное имя или IP рабочего места пользователя можно проверить статус службы LightManager, перезапустить ее, и, если надежда совсем умерла, переустановить LightManager.
2. Работа с почтовым сервером Zimbra
В случае использования почтового сервера Zimbra, первой линии ТП требуется добавлять адреса новых пользователей в общий список почтовых рассылок и в корпоративную адресную книгу. Все, что требуется, указать постовый адрес нового пользователя и нажать кнопку Add/Send  
3. Автоматизация настройк хостов для "удаленщиков" и внесение изменений на сетевом оборудовании под них
Функционал этого пункта применим для ситуаций, когда в организации выдаются на удаленную работу ноутбуки с ОС на базе Debian/Ubuntu, а подключение происходит по протоколам Wireguard/L2tp over IPSec к оборудованию вендора Mikrotik. Сервис позволяет установить пакеты для выбранного протокола подключения, Remina, Anydesk на ноутбук для удаленной работы, полностью сконфигурировать это ПО. Более широко фанкционал сервиса раскрываться только при конфигурировании настроек Wireguard, так как есть возможность заранее определить и зарезервировать выдаваемый ноутбуку при подключениик VPN IP-адреc, настроить режим работы сотрудника, разрешенный пул адресов для доступа из VPN-подсети, подкинуть на настраиваемый ноутбук скрипт для переключения на резервный endpoint при проблемах с основным.
